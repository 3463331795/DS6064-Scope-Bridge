# DS6064 Scope Bridge

Let Codex or another AI agent use a RIGOL DS6064 / DS6000 oscilloscope safely.

DS6064 Scope Bridge is a USB-TMC oscilloscope bridge for AI-assisted embedded debugging. It gives an AI agent a safe JSON CLI and a Codex skill, so a user can ask natural-language questions such as:

```text
用示波器读取 CH1、CH2、CH3 的波形，并分析三路 PWM 的时序关系。
```

The bridge then talks to the oscilloscope through PyVISA / NI-VISA, reads built-in measurements, captures waveforms, and writes an evidence package containing JSON, CSV, PNG, and manifest files.

<img src="docs/assets/cover.jpg" alt="RIGOL DS6064 running a three-channel PWM capture" width="820">

## What You Can Do

- Ask AI to read DS6064 measurements such as frequency, period, duty cycle, and Vpp.
- Capture CH1-CH4 waveforms into CSV and PNG files.
- Capture multi-channel timing evidence in one command with `snapshot`.
- Hand `manifest + CSV + PNG` to another AI for waveform or timing analysis.
- Keep oscilloscope access safer with JSON output, command allowlists, a file lock, and a watchdog.
- Use it as a Codex skill for natural-language workflows instead of manually sending SCPI.

This project is aimed at embedded developers who want AI to help with PWM, clock, CAN physical-layer checks, power ripple, ringing, overshoot, or general signal inspection.

## How It Works

```text
Natural-language request
-> Codex / AI agent
-> rigol-ds6064-scope skill
-> Python JSON CLI
-> PyVISA + NI-VISA
-> USB-TMC
-> RIGOL DS6064 / DS6000 oscilloscope
-> measurements / CSV / PNG / manifest
-> AI engineering analysis
```

The project intentionally focuses on USB-TMC. LAN/TCPIP control is not implemented.

## Requirements

- Windows host.
- Python 3.10+.
- NI-VISA installed and working.
- PyVISA Python dependencies from `requirements.txt`.
- A RIGOL DS6064 or compatible DS6000-series oscilloscope connected through the rear USB DEVICE port.

Before running Python commands, close Ultra Sigma, NI MAX, or any other tool that may already own the VISA session.

## Quick Start

Clone the repository and install dependencies:

```powershell
git clone https://github.com/3463331795/DS6064-Scope-Bridge.git
cd DS6064-Scope-Bridge
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
Copy-Item .env.example .env
```

Edit `.env` and set your own VISA resource:

```env
RIGOL_CONNECTION=USB
RIGOL_SCOPE_RESOURCE=USB0::0x1AB1::0x04B0::<YOUR_SERIAL>::INSTR
RIGOL_SCOPE_TIMEOUT_MS=20000
RIGOL_DEFAULT_CHANNEL=CHANnel1
RIGOL_CLEAR_ON_CONNECT=0
RIGOL_VISA_ACCESS_MODE=no_lock
RIGOL_VISA_LIBRARY=auto
RIGOL_CLI_TIMEOUT_MS=30000
RIGOL_LOCK_TIMEOUT_MS=5000
```

If you do not know the resource string yet, run:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py list
```

Then check the connection:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py probe-open --query-idn --open-timeout-ms 5000
.\.venv\Scripts\python.exe src\scope_cli.py health --channels CHANnel1 CHANnel2 CHANnel3 --points 64
```

## Common Commands

Read the scope identity:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py idn
```

Read built-in measurements from CH1:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py freq --channel CHANnel1
.\.venv\Scripts\python.exe src\scope_cli.py period --channel CHANnel1
.\.venv\Scripts\python.exe src\scope_cli.py duty --channel CHANnel1
.\.venv\Scripts\python.exe src\scope_cli.py vpp --channel CHANnel1
```

Capture one channel:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py capture --channel CHANnel1 --points 1200
```

Capture three channels into one CSV and one PNG:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py capture-multi --channels CHANnel1 CHANnel2 CHANnel3 --points 1200
```

Create the recommended AI handoff package:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py snapshot --channels CHANnel1 CHANnel2 CHANnel3 --points 1200
```

Read the latest saved manifest without touching hardware:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py latest
```

## Using It As A Codex Skill

This repository includes the skill in two places:

```text
SKILL.md
.agents/skills/rigol-ds6064-scope/SKILL.md
```

When Codex can see the repository, oscilloscope-related requests should trigger the `rigol-ds6064-scope` skill. The intended workflow is:

```text
User: 用示波器看一下 CH1 的 PWM 是否正常。
Codex: idn -> summary/freq/duty/vpp -> capture or snapshot -> analysis
```

For multi-channel work, prefer asking for a snapshot:

```text
读取 CH1、CH2、CH3 的波形，生成一份可以交给其他 AI 分析的证据包。
```

The skill is designed to call the wrapped CLI first. It should not send raw SCPI directly unless the user explicitly asks for low-level recovery and the command has been checked against the safety rules.

## JSON Output Contract

Every CLI command prints one JSON object.

Success:

```json
{"ok": true, "data": {}}
```

Failure:

```json
{"ok": false, "error": "message"}
```

AI agents should parse `ok` first and should not rely on stderr for machine decisions. File-producing commands return `csv_path`, `image_path`, and `manifest_path` as project-relative paths.

For more detail, see [docs/CLI_CONTRACT.md](docs/CLI_CONTRACT.md).

## Safety Rules

The normal AI workflow may read measurements, capture waveforms, save CSV/PNG/manifest files, and analyze saved data.

Avoid automatic use of configuration-changing commands. `autoscale` changes the oscilloscope state and should only be used when the user clearly allows it.

The CLI blocks dangerous SCPI patterns by default:

```text
*RST
:STOR
:SAVE
:LOAD
:DISK
:SYSTem:SECure
```

Do not run concurrent AI/tool calls against the same oscilloscope. Hardware-touching commands are guarded by a lock file under `outputs/logs/` and a watchdog timeout.

## Troubleshooting USB-TMC

USB-TMC is not a serial console. Common problems:

| Symptom | Likely Cause | What To Try |
|---|---|---|
| `list` cannot see the scope | Driver or cable issue | Check NI-VISA, USB DEVICE port, and Windows Device Manager |
| `list` sees it but `idn` hangs | Another tool owns the session | Close Ultra Sigma / NI MAX and replug USB |
| Waveform read times out | Too many points or short timeout | Start with `--points 1200`, keep timeout around 20000 ms |
| Random hangs | Concurrent calls or stale session | Run commands one at a time; rely on the CLI lock |
| Device disappears after idle | USB power saving | Disable Windows USB selective suspend |

If VISA resource opening is unstable, use:

```powershell
.\.venv\Scripts\python.exe src\scope_cli.py probe-open --query-idn --open-timeout-ms 5000
```
